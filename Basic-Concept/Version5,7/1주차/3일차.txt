※안드로이드 최근 버전의 주요 특징
- 젤리빈에서는 아이스크림 샌드위치 버전보다 더 빠르고 부드러운 그래픽에 중점을 두었다.

- 킷캣에서는 좀 더 빠른 그래픽과 웹 브라우징에 중점을 두었으며, 앱에서 콘텐츠를 직접 프린트할 수 있는 기능을 추가하였다.

- 롤리팝에서는 사용자가 보는 화면의 다자인이 완전히 새롭게 바뀌었고, 새로운 ART 런타임을 적용하고 그래픽이나 웹뷰의 성능을 개선함으로써 한 단계 더 업그레이드된 
안드로이드를 느낄 수 있도록 하는 데 중점을 두었다.

- 마시멜로에서는 실행할 때 사용자가 앱의 권한을 부여할 수 있도록 하고, 지문 인식 기능을 사용할 수 있도록 함으로써 보안을 강화하느데 중점을 두었다.

- 누가 버전에서는 멀티 윈도우를 지원하고 알림 기능을 확장함으로써 사용자가 더 편하고 빠르게 스마트폰을 사용할 수 있도록 하는 데 중점을 두었다.

- 오레오 버전에서는 PIP 모드를 지원하고 알림 기능이나 설정 기능을 좀 더 편리하게 만들었다.


※폴더들의 특징들
- /java: 자바 소스 파일이 있는 폴더 (/app/src/main)

- /res: XML 레이아웃, 그림 파일, 문자열을 정의한 XML 파일 등이 들어있다. (/app/src/main)

- /build: 개발 환경이 자동으로 만들어 내는 소스 파일이 들어가는 폴더이다.

- /assets: 이 폴더에 넣어서 처리하는 파일들을 저장함(예: 동영상 파일)


※소스 파일이 프로젝트로 묶여 빌드되기까지
- gradle이라는 것을 이용하여 빌드 과정을 자동으로 진행하며 코드나 리소스 정보가 바뀔 때마다 자동으로 빌드 결과를 만들어 낸다.

- 자바 코드 + 리소스 + 메니페스트 == 하나의 앱

- 하나의 앱이 만들어지면 런타임에서 실행될 수 있는 포맷으로 변환한다.(apk)

- 안드로이드 스튜디오에서 앱 작성 --> 자바에서 빌드(컴파일) --> 바이트코드로 변환 및 패키징 --> ART 런타임 (안드로이드 커널(Linux)
      <프로젝트 구성>		 <javac complier><dxtool>	     <aapt tool>


※gradle의 설정
- 다른 사람이 만든 프로젝트를 안드로이드 스튜디오에서 열다 보면 소스 파일에는 에러가 없는데 gradle설정 때문에 빌드되지 않는 경우가 있다.
이것은 Gradle Scripts 폴더 안에 있는 build.gradle 파일의 설정 정보 때문이다.

- compileSdkVersion: 컴파일할 때에 사용할 SDK 버전을 지정한다. (예를 들어 26이라면 API 버전이 26이라는 것이다.)

- minSdkVersion: 프로젝트가 지원하는 최소 SDK 버전을 지정한다. (예를 들어 값이 15로 되어있다면 API 버전이 15 이전의 단말에서는 앱이 동작하지 않는다는 뜻이다.)

- targetSdkVersion: 실행을 위해 검증한 SDK가 어떤 것인지를 지정한다.(예를 들어 값이 26이더라도 하위 버전인 24, 22 버전에서는 동작하지 않는 것은 아니다. 하지만 SDK 매니저에 API 버전이 26인 플랫폼이 설치 되어야 한다.)

- 현재 사용하고 있는 PC의 SDK 매니저에서 설치한 모듈의 버전과 프로젝트의 위 정보들이 서로 일치해야 한다.
